JavaScript Basics

What is JavaScript?

JavaScript is a client side scripting language used to improve the usability and functionality of websites and web applications.A simple example of this is client-side validation.It provides all the basic language elements that you'd find in any programming language:  variables, arrays, objects, control flow, arithmetic operations, boolean operations, functions, etc.

Consider a scenario where we have to validate a form which is incorrectly filled.Without client side validation, a developer would have to process the contents of a form via a web server, and then send a response back if the form was completed incorrectly. JavaScript can eliminate this by simply validating the data directly in a web browser. Common usage of JavaScript is listed below
  • Validate Forms – Ensure all required fields are entered, Ensure fields are entered as per specific format etc.
  • Create Cookies – Stores and retrieves information on a visitor’s computer.
  • JavaScript can enhance the interfaces HTML gives us.
  • You can use JavaScript to animate elements on a page — for example to show and hide information, or highlight specific sections of a page — this can make for a more usable, richer user experience.

Equality Operators

standard equality operators

The standard equality operators (== and !=) compare two operands without regard to their type. If either operand is a number or a boolean, the operands are converted to numbers if possible; else if either operand is a string, the other operand is converted to a string if possible. If both operands are objects, then JavaScript compares internal references which are equal when operands refer to the same object in memory.
    Equal (==)
      If the two operands are not of the same type, JavaScript converts the operands then applies strict comparison.
       123 == "123"
       true == 1
        Not equal (!=)
          Returns true if the operands are not equal. If the two operands are not of the same type, JavaScript attempts to convert the operands to an appropriate type for the comparison.
           var1 != 4
           var1 != "5"

          strict  equality operators

          The strict equality operators (=== and !==) perform equality comparisons on operands of the same type.

          Strict equal (===)
          Returns true if the operands are strictly equal with no type conversion.
          123 == "123"

          Strict not equal (!==)
          Returns true if the operands are not equal and/or not of the same type.
          3 !== '3'

          Console.log

          I have also came accross Console.log recently. We can view any messages logged to the console if you use a tool such as Firebug to inspect your code (or whichever tool you decide to use to inspect your code). This is particularly useful when you want to see if a function is executing, or if a variable is being passed/assigned properly or  just for figuring out what went wrong with your code.

          Advantages of using Console.log

          • it does not halt processes like alert does
          • It is non-blocking
          • You can log objects for instance, and then inspect them.
          • you can see what line of what script threw the log entry without putting the line into your message
          • if you have more than one thing you're debugging it can get real annoying to keep hitting 'ok' on your alert boxes.
